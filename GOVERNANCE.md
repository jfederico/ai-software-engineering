# Governance

## Project Ownership

This repository is maintained by project maintainers listed in the repository settings.

## Decision Model

- Minor wording and documentation fixes can be merged after one maintainer review.
- Methodology structure changes (phases, gates, required artifacts) require maintainer consensus.
- Breaking changes require a documented rationale and a version bump per VERSIONING.md.

## Proposal Process

1. Open an issue describing the change and expected impact.
2. Label it as enhancement or breaking-change.
3. Collect feedback from maintainers and contributors.
4. Implement via PR with migration notes if needed.

## Conflict Resolution

When maintainers disagree, default to:
- preserving backward compatibility
- prioritizing clarity and reproducibility
- delaying breaking changes to a major release
