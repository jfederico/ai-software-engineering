# Contributing

Thank you for contributing to AI Software Engineering Methodologies.

## Scope

This repository contains two methodology systems:
- Requirement Gathering and Documentation
- Phase-Gated Delivery

Contributions should improve clarity, reproducibility, and practical usability across projects.

## Ways to Contribute

- Improve prompt quality and clarity
- Improve methodology steps and guardrails
- Add examples and practical usage patterns
- Fix typos, structure, and broken links
- Propose better templates and gate criteria

## Contribution Workflow

1. Open an issue describing the problem or proposal.
2. Fork and create a focused branch.
3. Make changes with clear commit messages.
4. Open a pull request linked to the issue.
5. Address review feedback and keep scope tight.

## Content Guidelines

- Keep language concrete and actionable.
- Prefer short sections, checklists, and examples.
- Keep methodologies tool-agnostic unless explicitly labeled.
- Do not add proprietary or confidential examples.
- Preserve cross-links between top-level guides and prompts.

## Commit Guidance

Use conventional style where possible:
- feat: new capability
- fix: correction
- docs: documentation-only changes
- refactor: structure improvement without behavior change
- chore: tooling or repo maintenance

## Pull Request Checklist

- Change has clear purpose and scope.
- Related docs updated where needed.
- Internal links verified.
- No sensitive data included.
- Changelog updated for user-visible changes.

## Review Expectations

Maintainers prioritize:
- correctness
- clarity
- consistency
- practical reproducibility
